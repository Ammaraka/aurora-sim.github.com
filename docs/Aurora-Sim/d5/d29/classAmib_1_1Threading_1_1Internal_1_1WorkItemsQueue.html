WorkItem Amib.Threading.Internal.WorkItemsQueue.DequeueWorkItem ( int  millisecondsTimeout,
WaitHandle  cancelEvent 
)

Waits for a work item or exits on timeout or cancel.

Parameters:
millisecondsTimeoutTimeout in milliseconds
cancelEventCancel wait handle
Returns:
Returns true if the resource was granted

This method cause the caller to wait for a work item. If there is at least one waiting work item then the method returns immidiately with true.

If there are no waiting work items then the caller is queued between other waiters for a work item to arrive.

If a work item didn't come within millisecondsTimeout or the user canceled the wait by signaling the cancelEvent then the method returns false to indicate that the caller didn't get a work item.
